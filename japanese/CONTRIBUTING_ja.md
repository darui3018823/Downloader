# コントリビューションガイド

> 📖 **言語:** [English](../CONTRIBUTING.md) | [日本語](./CONTRIBUTING_ja.md)

Video Downloaderへの貢献に興味を持っていただきありがとうございます!どなたからの貢献も歓迎します。

## 貢献方法

### バグ報告

バグを見つけた場合は、以下の情報を含むissueを作成してください:
- 明確でわかりやすいタイトル
- 問題を再現する手順
- 期待される動作
- 実際の動作
- 環境情報(OS、Rustバージョン、yt-dlpバージョン)
- 関連するログやエラーメッセージ

### 機能追加の提案

機能追加の提案を歓迎します!以下を含むissueを作成してください:
- 明確でわかりやすいタイトル
- 提案する機能の詳細な説明
- なぜこの機能が役立つのか
- どのように動作するかの例

### プルリクエスト

1. **リポジトリをフォーク**して`main`からブランチを作成
2. **変更を加える** - コードガイドラインに従ってください
3. **変更を十分にテスト**
4. **明確でわかりやすいコミットメッセージでコミット**
5. **フォークにプッシュ**してプルリクエストを提出

#### プルリクエストのガイドライン

- 変更を集中させる - 1つのPRに1つの機能または修正
- 機能変更があればドキュメントを更新
- 新機能にはテストを追加
- すべてのテストが通ることを確認
- 既存のコードスタイルに従う
- わかりやすいコミットメッセージを書く

## 開発環境のセットアップ

### 前提条件

- Rust 1.70以上
- Git

### ソースからビルド

```bash
# リポジトリをクローン
git clone <repository-url>
cd Downloader

# デバッグモードでビルド
cargo build

# リリースモードでビルド
cargo build --release

# テストを実行
cargo test

# アプリケーションを実行
cargo run
```

### コードスタイル

このプロジェクトは標準的なRustの規約に従っています:
- `cargo fmt`でコードをフォーマット
- `cargo clippy`で一般的なミスをチェック
- 明確で自己文書化されたコードを書く
- 複雑なロジックにはコメントを追加
- Rustの命名規則に従う

### テスト

- 新機能には単体テストを追加
- `cargo test`で既存のすべてのテストが通ることを確認
- 異なるプラットフォーム(YouTube、Twitch、Twitter/X)で手動テスト
- エッジケースとエラー条件をテスト
